# 需求文档：挪车通知Android App

## 基本信息
- **需求名称**：挪车通知Android App（Parking Notification App）
- **需求ID**：REQ-PNA-001
- **创建时间**：2026-06-29
- **需求来源**：用户直接提出

## User Story
作为 车主/驾驶员
我想要 当我的车阻碍他人通行时，能通过手机短信实时收到挪车通知并强提醒
以便于 能及时挪车，避免影响他人，同时不需要在挡风玻璃上留下个人电话号码

## 功能需求

### 1. 核心功能

#### 1.1 通知项管理
- [ ] **创建通知项**：用户可创建新的通知项，每个通知项包含：
  - 通知项名称（用户自定义）
  - 关联的触发规则（从规则列表选择或新增）
  - 提醒方式配置（长震动开关 + 系统提示音选择）
  - 启用/禁用状态
- [ ] **通知项列表**：以卡片列表展示所有已创建的通知项，显示摘要信息（名称、规则数、启用状态）
- [ ] **编辑通知项**：支持修改通知项名称、触发规则、提醒方式
- [ ] **删除通知项**：支持删除不需要的通知项，删除前需二次确认
- [ ] **启用/禁用通知项**：支持在不删除的情况下临时禁用某个通知项

#### 1.2 触发规则管理
- [ ] **规则列表选择**：在创建/编辑通知项时，支持从已创建的触发规则列表中选择已有规则
- [ ] **快速新增规则**：支持在通知项编辑界面直接新增触发规则，新增后自动更新到全局规则列表
- [ ] **规则必录项**：
  - 短信手机号：支持模糊匹配
  - 短信内容关键词：支持模糊匹配
  - 默认匹配模式：模糊匹配（LIKE）
- [ ] **规则列表管理**：独立的规则管理页，支持查看/编辑/删除所有已创建的触发规则
- [ ] **匹配逻辑**：当收到短信时，短信发送号码包含规则中设置的手机号片段 **且** 短信内容包含规则中设置的关键词片段，即判定为匹配成功

#### 1.3 提醒方式配置
- [ ] **长震动提示**：开关控制，开启后触发提醒时手机持续长震动（非短促震动），直到用户手动关闭
- [ ] **系统提示音选择**：调用Android系统通知铃声选择器，用户可从系统铃声列表中选择提示音
- [ ] **预览测试**：支持在配置界面点击预览按钮，测试当前选择的震动和铃声效果

#### 1.4 短信监听与规则匹配引擎
- [ ] **短信接收广播监听**：注册 `BroadcastReceiver` 监听 `SMS_RECEIVED_ACTION` 或使用 `Telephony.Sms.Inbox` ContentObserver（Android 14+适配）
- [ ] **后台常驻服务**：使用 `ForegroundService` 保持后台运行，持续监听新短信
- [ ] **规则匹配引擎**：收到短信后，逐条匹配所有启用状态的通知项及其关联规则
  - 手机号匹配：短信发送号码包含规则的手机号片段（不区分大小写）
  - 内容匹配：短信正文包含规则的内容关键词片段
  - 多个规则时，任一规则满足即触发
- [ ] **匹配成功后触发提醒**：根据匹配到的通知项配置的提醒方式，触发长震动 + 系统提示音

#### 1.5 通知栏常驻提醒
- [ ] **触发时发送通知栏提醒**：规则匹配成功后，立即向Android系统通知栏发送一条高优先级（importance=HIGH）的常驻通知
- [ ] **通知栏展示内容**：通知栏展示匹配到的短信发送号码、短信内容摘要、触发时间
- [ ] **通知不可滑动清除**：该通知为 `ongoing` 类型，用户无法侧滑清除，仅在app内点击关闭按钮后才移除
- [ ] **点击通知进入关闭页**：用户点击通知栏提醒后，打开app的全屏提醒关闭页面
- [ ] **通知栏与全屏联动**：通知栏常驻通知与全屏提醒页面保持状态同步，任一方式关闭都会同步移除通知栏提醒和终止震动铃声

#### 1.6 提醒关闭交互
- [ ] **全屏提醒页面**：匹配成功后，自动弹出全屏提醒Activity（即使app在后台或被关闭）；同时通知栏常驻通知同步展示
- [ ] **大按钮关闭**：页面中央展示醒目的"关闭提醒"大按钮（占屏幕宽度60%+），点击后终止震动和铃声，并移除通知栏常驻通知
- [ ] **提醒锁定**：在提醒状态下，按返回键或Home键不会关闭提醒，保持前台展示
- [ ] **提醒详情展示**：提醒页面同时展示匹配到的短信发送号码、短信内容片段、触发的时间
- [ ] **通知栏点击进入**：若用户未在全屏页面（如锁屏状态未解锁），可通过点击通知栏常驻通知进入全屏关闭页面

#### 1.7 提醒历史记录
- [ ] **历史记录列表**：按时间倒序展示所有已触发的提醒记录
- [ ] **历史详情**：每条记录展示触发时间、短信发送号码、短信内容、匹配的规则、关闭时间
- [ ] **历史清理**：支持单条删除和批量清空

### 2. 系统权限管理

#### 2.1 短信读取权限（RECEIVE_SMS / READ_SMS）
- [ ] **权限检测**：创建/保存通知项时检测是否已授予短信权限
- [ ] **权限引导弹窗**：未授予时弹出系统权限引导弹窗，说明需要该权限的原因
- [ ] **权限导航**：引导用户跳转到系统权限设置页
- [ ] **权限持续引导**：弹窗只有用户主动点击"关闭"才会消失，拒绝后再次检测到未授权会重新弹出

#### 2.2 通知权限（POST_NOTIFICATIONS）
- [ ] **权限检测**：创建/保存通知项时检测是否已授予通知权限
- [ ] **权限引导弹窗**：未授予时弹出引导弹窗（与短信权限合并引导或分步引导）
- [ ] **权限导航**：引导用户跳转到系统通知权限设置页

#### 2.3 后台运行权限
- [ ] **权限检测**：app启动时检测是否已开启后台运行权限/电池优化白名单
- [ ] **厂商适配引导**：针对华为（EMUI/HarmonyOS）、小米（MIUI/HyperOS）、OPPO（ColorOS）、vivo（OriginOS）等主流国产ROM，提供各自的后台权限设置引导流程
- [ ] **通用引导**：非国产ROM设备提供通用的电池优化白名单设置引导
- [ ] **引导弹窗**：同权限引导逻辑，弹窗只有用户主动点击关闭才会消失

### 3. 数据需求
- **存储方式**：本地SQLite数据库（Room）
- **数据表设计**：
  - `notification_items`：通知项表（id, name, enable_vibration, ringtone_uri, ringtone_name, enabled, created_at, updated_at）
  - `trigger_rules`：触发规则表（id, phone_keyword, content_keyword, created_at, updated_at）
  - `notification_item_rules`：通知项-规则关联表（item_id, rule_id）
  - `notification_history`：提醒历史表（id, item_id, sender_number, message_content, matched_rule_id, triggered_at, dismissed_at, dismissed_by）
- **数据安全**：所有数据仅存储在本地，不上传网络

### 4. 用户界面需求

#### 4.1 主页面（通知项列表）
- 顶部：app名称 + 当前运行状态指示器（服务运行中/已停止）
- 中部：通知项卡片列表，每个卡片显示名称、规则数量、启用开关
- 底部：FAB按钮创建新通知项
- 导航：底部导航栏或侧边栏，包含"通知项"、"触发规则"、"提醒历史"、"设置"

#### 4.2 通知项编辑页
- 通知项名称输入框
- 提醒方式配置区（震动开关 + 铃声选择 + 预览）
- 触发规则配置区（已选规则列表 + 添加规则按钮）
- 保存按钮

#### 4.3 触发规则管理页
- 规则列表（手机号关键词 + 内容关键词）
- 新增/编辑/删除规则操作

#### 4.4 全屏提醒页
- 背景：高亮警示色（红色/橙色）
- 中央：大号"关闭提醒"按钮
- 信息：发送号码、短信内容片段、触发时间
- 注意：该页面由通知栏常驻通知点击触发，两者联动

#### 4.5 通知栏常驻通知样式
- 渠道：创建独立的 `NotificationChannel`（channel_id: "parking_alert", 名称: "挪车提醒"）
- 优先级：IMPORTANCE_HIGH，支持弹出通知栏横幅
- 图标：自定义通知图标（车辆/警示相关）
- 样式：`BigTextStyle` 展开显示完整短信内容和触发时间
- 动作：点击 PendingIntent 打开全屏提醒关闭页面

#### 4.6 设置页
- 服务状态控制
- 权限管理入口
- 厂商后台引导入口
- app版本信息

## 验收标准（AC）

### AC1: 创建通知项成功
**Given** 用户已打开app
**When** 用户填写通知项名称，配置震动和铃声，添加触发规则后点击保存
**Then** 通知项创建成功，出现在通知项列表中，状态为启用

### AC2: 触发规则模糊匹配 - 匹配成功
**Given** 通知项已启用，规则设置为手机号含"138"且内容含"挪车"
**When** 收到来自"13800138000"的短信，内容为"您的车挡道了请挪车"
**Then** 规则匹配成功，触发震动+铃声提醒

### AC3: 触发规则模糊匹配 - 内容不匹配
**Given** 通知项已启用，规则设置为手机号含"138"且内容含"挪车"
**When** 收到来自"13800138000"的短信，内容为"快递到了请查收"
**Then** 规则匹配失败，不触发提醒

### AC4: 触发规则模糊匹配 - 号码不匹配
**Given** 通知项已启用，规则设置为手机号含"138"且内容含"挪车"
**When** 收到来自"13900139000"的短信，内容为"请挪车"
**Then** 规则匹配失败，不触发提醒

### AC5: 通知栏常驻提醒展示
**Given** 规则匹配成功，提醒已触发
**Then** 系统通知栏展示一条常驻通知，显示发送号码和短信摘要，用户无法侧滑清除

### AC6: 通知栏点击进入全屏关闭页
**Given** 通知栏常驻提醒正在展示
**When** 用户点击该通知
**Then** 打开app全屏提醒关闭页面，展示大按钮"关闭提醒"

### AC7: 用户关闭提醒（全屏页面或通知栏联动）
**Given** 提醒已触发，全屏提醒页面已展示，通知栏常驻通知已展示
**When** 用户点击全屏页面"关闭提醒"按钮
**Then** 震动停止，铃声停止，全屏页面关闭，通知栏常驻通知同步移除，历史记录中记录关闭时间

### AC8: 权限引导弹窗 - 拒绝短信权限
**Given** 用户创建通知项时未授予短信读取权限
**When** 点击保存后弹出权限引导弹窗，用户点击"关闭"关闭弹窗
**Then** 通知项保存成功但处于禁用状态，下次打开通知项编辑页再次检测权限

### AC9: 权限引导弹窗 - 授予权限
**Given** 用户创建通知项时未授予短信读取权限
**When** 点击保存后弹出权限引导弹窗，用户点击"去设置"并授予权限后返回
**Then** 通知项保存成功且处于启用状态

### AC10: 后台运行引导
**Given** app检测到未开启后台运行权限
**When** app启动时弹出后台权限引导弹窗
**Then** 弹窗显示当前设备厂商的后台权限设置步骤引导，用户可点击"去设置"跳转

### AC11: 提醒历史记录
**Given** 已有多次触发提醒的记录
**When** 用户进入"提醒历史"页面
**Then** 按时间倒序展示所有提醒记录，包含发送号码、短信内容、触发时间等信息

### AC12: 编辑通知项
**Given** 通知项已存在
**When** 用户点击编辑，修改名称/提醒方式/规则后保存
**Then** 通知项更新成功，列表显示更新后的信息

### AC13: 多通知项独立匹配
**Given** 用户创建了两个通知项，规则不同
**When** 收到短信仅匹配其中一个规则
**Then** 只有匹配的通知项触发提醒

### AC14: 返回键不关闭提醒
**Given** 全屏提醒页面正在展示
**When** 用户按下返回键
**Then** 提醒不被关闭，持续震动和铃声

## 非功能需求

### 性能要求
- **短信匹配延迟**：从收到短信到触发提醒，延迟不超过3秒
- **后台内存占用**：ForegroundService内存占用控制在50MB以内
- **启动时间**：app冷启动时间不超过2秒
- **列表加载**：通知项列表和历史列表加载不超过500ms（100条以内数据）

### 安全要求
- **数据传输**：所有短信数据仅本地处理，不上传网络或第三方服务
- **数据存储**：使用Android Keystore对本地数据库进行加密
- **权限最小化**：仅在用户创建通知项并启用后才请求短信权限
- **隐私保护**：应用描述和隐私政策中明确声明短信数据的用途仅限于本地规则匹配

### 可用性要求
- **后台服务存活**：在主流设备上，服务应在启动后持续运行，不被系统自动杀死
- **厂商兼容引导**：覆盖华为、小米、OPPO、vivo四大国产ROM的后台设置引导
- **系统版本兼容**：支持Android 8.0（API 26）至最新Android版本
- **低功耗运行**：后台服务采用JobScheduler/WorkManager + 低功耗模式，减少CPU和网络消耗

### 兼容性要求
- **Android版本**：兼容 Android 8.0+（API 26+）
- **屏幕适配**：支持主流屏幕尺寸（5.0寸 - 7.0寸），适配折叠屏
- **语言**：简体中文（默认），后续可扩展英文

## 约束条件
- **技术约束**：使用Kotlin + Jetpack Compose开发，数据库使用Room
- **平台约束**：仅限Android平台
- **合规约束**：需遵守《个人信息保护法》，短信数据仅本地处理
- **时间约束**：无硬性交付时间要求

## 依赖关系
- **Android系统API**：
  - `android.permission.RECEIVE_SMS`
  - `android.permission.READ_SMS`（Android 14+替代方案）
  - `android.permission.POST_NOTIFICATIONS`（Android 13+）
  - `android.permission.FOREGROUND_SERVICE`
  - `android.permission.FOREGROUND_SERVICE_DATA_SYNC`（Android 14+）
  - `android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`
  - `NotificationChannel`（Android 8.0+ 通知渠道常驻提醒）
- **第三方库**：
  - Room（本地数据库）
  - Jetpack Compose（UI框架）
  - WorkManager（后台任务调度）

## 风险评估

| 风险项 | 风险等级 | 应对措施 |
|--------|---------|---------|
| Android 14+ 短信权限收紧（READ_SMS受限） | 高 | 使用 `Telephony.Sms.Inbox` ContentObserver + 用户手动授权；备选方案：使用NotificationListenerService监听短信通知 |
| 国产ROM后台进程被杀死 | 高 | 实现厂商自启动/后台白名单引导流程；使用ForegroundService + 低优先级通知保持存活 |
| 多个通知项同时匹配导致多重提醒 | 中 | 设计提醒合并逻辑：同一时间内多个匹配仅触发一次提醒，关掉全部 |
| 短信数据隐私安全 | 中 | 仅本地处理不联网，隐私政策明确声明，使用加密存储 |
| 系统铃声选择器在不同ROM表现不一致 | 低 | 使用系统标准 `RingtoneManager.ACTION_RINGTONE_PICKER`，适配各ROM |
