# 设计Spec: 挪车通知 Android App

## 项目概述
**挪车通知**是一款Android工具类App，核心功能是实时监听手机短信，当收到匹配特定关键词（如"挪车"、"挡道"）的短信时，触发强提醒（长震动+系统提示音+全屏提醒+通知栏常驻提醒），让车主能及时挪车，同时无需在挡风玻璃上留电话号码。

## 目标受众
- 中国车主/驾驶员，日常在市区临时停车
- 年龄25-50岁，实用主义用户
- 需要app"在关键时刻绝对可靠"，日常使用频率低但关键时刻不能掉链子

## 情感基调
- **日常状态**：干净、可靠、专业、不打扰——app在后台安静运行
- **提醒触发时**：紧急、醒目、不可忽视——全屏红色警示+长震动+响亮铃声
- **品牌气质**：值得信赖的实用工具，不花哨，不廉价

## 6个核心界面

### 1. 通知项列表（主页面）
- 顶部：app名称 + 服务运行状态指示器（运行中🟢/已停止🔴）
- 中部：通知项卡片列表，每个卡片显示名称、关联规则数量、启用/禁用开关
- 底部：FAB圆形按钮创建新通知项
- 底部导航栏（4个Tab）：通知项、触发规则、提醒历史、设置

### 2. 通知项编辑页
- 通知项名称输入框（顶部）
- 提醒方式配置区：震动开关 + 铃声选择按钮 + 预览测试按钮
- 触发规则配置区：已选规则列表 + "添加规则"按钮
- 底部：保存按钮

### 3. 触发规则管理页
- 规则列表：每条显示手机号关键词 + 内容关键词
- 每条规则可编辑/删除
- 底部FAB新增规则

### 4. 全屏提醒页（最关键页面）
- 背景：高亮警示色（红色/橙色系）
- 中央：超大"关闭提醒"按钮（占屏幕宽度60%+）
- 信息展示区：发送号码、短信内容摘要片段、触发时间
- 行为：按返回键不关闭，Home键不关闭 — 用户必须点击"关闭提醒"按钮
- 顶部：可选的简短说明文字"您的车辆可能阻碍了他人通行"

### 5. 提醒历史页
- 按时间倒序展示所有提醒记录
- 每条展示：触发时间、发送号码、短信内容摘要、关闭时间
- 支持单条删除和批量清空

### 6. 设置页
- 服务运行状态 + 开关控制
- 权限状态检测入口（短信权限、通知权限）
- 厂商后台白名单引导入口（华为/小米/OPPO/vivo）
- App版本信息

## 输出格式
- **交互式HTML原型**，单文件 inline React + Babel
- **平铺多台Android手机并排**，每台可交互
- 每台手机是一个独立状态机，初始落在不同主界面（共6台）
- 每台手机上方标注界面名称
- 手机框：仿Android 14风格（圆角矩形+前置摄像头挖孔+简洁边框）

## 设计约束
- 无品牌色/Logo/Design System（从零设计）
- 界面语言：简体中文
- 屏幕尺寸参考：Android主流手机 1080×2340 左右
- 色彩限制：日常界面不要用紫色渐变（AI slop），提醒页可以用红/橙

## 图片需求
- 纯UI原型，无需内容图片
